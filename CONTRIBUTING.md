# Contributing

Work in small vertical slices. Prefer changes that strengthen a traceable path from source data to obligation, evidence, approval, or value.

Before opening a PR or handing off work:

1. Explain the domain boundary touched.
2. List commands run and their result.
3. Note any generated files intentionally excluded.
4. Update docs or ADRs when changing architecture.

Do not mix unrelated refactors with feature work.
