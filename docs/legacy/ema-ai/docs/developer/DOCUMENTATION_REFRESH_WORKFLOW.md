# Documentation Refresh Workflow

## Standard Procedure
1. Audit existing docs and `.ai` memory.
2. Identify stale or conflicting claims.
3. Map status as Implemented / Partial / Planned / Future.
4. Update docs, diagrams, and paper sections.
5. Refresh `docs/DOCUMENTATION_INDEX.md`.
6. Update `.ai` memory files.
7. Run docs validation checks.
8. Stage docs/memory only; exclude generated artifacts and real project data.

## Guardrails
- No secrets or proprietary client files.
- No production-readiness overclaims.
- No claims that AI/SEION modifies official readiness.
