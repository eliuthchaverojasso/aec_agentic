\section{What EMA AI Does}
EMA AI is a Revit-first engineering intelligence pilot that checks Owner Requirements against available project evidence. It begins in Revit, uses the synced model as the deterministic source of truth, and produces an HTML report for review. The report explains what the model appears to satisfy, what still needs human judgment, and what information is missing. EMA AI assists engineering review; it does not certify compliance or approve final design decisions.

\subsection*{Pilot scope}
The first testing cycle should use the shared baseline project \textbf{MEP-NISD-MIDDLE SCHOOL}. The corresponding workbook in the repository evidence set is \code{NORTHWEST ISD 06.02.2025.xlsx}. Initial tuning focus is requirement classification, evidence selection, status assignment, confidence, urgency, and recommended next action. Ask EMA AI is secondary during the first cycle.

\subsection*{What is in scope}
\begin{itemize}
  \item Revit add-in
  \item Owner Requirements loading
  \item Model synchronization
  \item Deterministic requirement check
  \item HTML report
  \item Requirement-level traceability
  \item Deterministic fallback
\end{itemize}

\subsection*{What is optional}
\begin{itemize}
  \item Full local platform with backend API, PostgreSQL, and dashboard
  \item Local AI via Ollama
\end{itemize}
