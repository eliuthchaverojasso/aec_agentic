# Project Setup Workflow

## Purpose

Describes how users create or bind a project, connect it to a landing folder, and establish client context.

## Steps

1. Pick or create a project.
2. Bind or create a client.
3. Select a landing project folder.
4. Review detected files and client suggestion.
5. Save the binding.
6. Refresh Requirements, Documents, and Readiness.

## Caveats

- Client binding is explicit.
- Owner requirements need a client link before they can become meaningful.
