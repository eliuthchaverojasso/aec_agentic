# Risks

- Accidentally importing real project data into source.
- Refactoring migrated EMA code before baseline tests are re-established.
- Letting AI outputs appear authoritative.
- Creating generic shared utilities that couple all domains.
- Replacing source-system authority instead of linking to it.
- Running missions without budgets, leases, or completion criteria.
- Embedding secrets, client files, or raw private outputs into memory indexes.
- Letting proposed self-improvements alter canonical skills without tests and review.
